# NeoOptimize Adaptive Configuration

## Product Goal
NeoOptimize is a modular Windows optimization app with adaptive UI.
Core principle: show simple actions first, reveal advanced controls on demand.

## Core Modules

### Dashboard
- Show real-time CPU, RAM, Disk, Network usage.
- Show health status: green/yellow/red.
- Provide quick actions:
  - Smart Clean
  - Smart Optimize
  - Smart Fix
- Integrate Log Center preview.

### Cleaner
- Default mode: cache, temp, junk cleanup only.
- Advanced mode:
  - Registry cleanup
  - Driver leftovers cleanup
  - Bloatware removal
- All runs must write execution logs.

### Optimizer
- Smart Mode: choose optimization based on current system pressure.
- Manual Mode:
  - CPU tuning
  - RAM tuning
  - Disk tuning
  - Network tuning
- Privacy Pack:
  - telemetry reduction
  - background app restrictions
  - tracking-related toggles

### System Tools
- System Repair bundle:
  - SFC
  - DISM
- Safety controls:
  - create restore point before major action
  - registry backup before risky operations
- Utilities:
  - disk check helper
  - event viewer shortcut

### Security
- Unified scan profile:
  - ClamAV
  - KicomAV
  - Defender toggle
- Firewall and telemetry presets:
  - Basic
  - Advanced
- USB autorun blocker default enabled.

### Settings
- Experience mode:
  - Simple
  - Advanced
- Theme options:
  - Dark
  - Light
- Language options:
  - Indonesian
  - English
- Scheduler:
  - auto clean
  - auto optimize

## Adaptive Behavior

### UI Modes
- Simple mode:
  - expose only Dashboard quick actions and key status cards.
- Advanced mode:
  - expose all modules and advanced controls.

### Mini Tray AI Rules
- Trigger when resource thresholds are exceeded:
  - CPU > 80 percent: prioritize RAM release.
  - RAM > 85 percent: run cache flush.
  - High disk IO: run light cleanup.
  - High network latency: run DNS flush.
- On startup:
  - run lightweight health scan.
- Before shutdown:
  - optional pre-shutdown cleanup if enabled.

## Logging Contract
- Every action produces a structured log entry with:
  - timestamp
  - module
  - action
  - result
  - metrics (if available)
- Log report target:
  - report-dd-mm-yyyy.html
- Log Center supports:
  - filter by module (Cleaner, Optimizer, Security, System)
  - delete selected report
  - export report

## Architecture Mapping
- UI layer:
  - Dashboard, Cleaner, Optimizer, SystemTools, Security, Settings, Support
- Core layer:
  - Cleaner engine
  - Optimizer engine
  - System tools engine
  - Security engine
  - Log manager
  - Scheduler
  - Tray service
- Service layer:
  - update service
  - optional remote assist
  - localization service

## Release Readiness
- App should work offline-first.
- Advanced actions should request elevated privilege when required.
- Always write logs for traceability and diagnostics.
